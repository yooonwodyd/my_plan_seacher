# AI 개인 비서 최종 설계 문서

> 작성 기준: 2026-04-20
> 환경: iPhone / iPad / MacBook / Windows 데스크톱 (24시간 상시 가동) → 추후 Mac Mini 이전 예정
> 저장소: Notion (일정표, 가계부 모두)
> 알림: Notion Calendar 앱 (확정 일정 자동 알람) + ntfy 푸시 (AI 판단 스마트 알람)

---

## 1. 개요

### 1-1. 목적 & 핵심 가치

사용자는 의도만 말하고, AI가 다음을 대신한다.

1. **목표 → 일정 자동 생성** — 큰 목표를 하위 작업으로 쪼개고, 과거 기록 기반으로 소요 시간을 추정해 빈 시간에 세션 단위로 배치한다.
2. **일정 변경 시 자동 재조율** — 새 일정이 생기거나 기존 일정이 밀리면, 영향받는 일정들을 마감일과 우선순위를 고려해 자동으로 재배치 제안한다.
3. **결제 자동 가계부** — 결제 발생 시 영수증/문자를 파싱해 Notion 가계부에 자동 기록한다.
4. **상황 인식 알람** — 일정 수행 전 AI가 내용을 판단해 "짐 챙기세요", "준비물 확인" 같은 맥락 알람이 필요한지 스스로 판단하고, 필요하면 사용자에게 먼저 묻거나 자동 발송한다.
5. **승인 기반 실행** — 모든 Notion 변경은 사용자 승인 후에만 적용한다. 저장된 `확정` 일정은 Notion Calendar를 통해 자동 알람이 울린다.

"해야 하는 일"이 "언제, 어떻게 할지"로 자동 변환되는 것이 핵심 가치다.

### 1-2. 환경

| 항목 | 내용 |
|---|---|
| 저장소 | Notion (일정, 가계부) |
| 서버 | Windows 데스크톱 (상시 가동) → 추후 Mac Mini |
| 입력 | iPhone/iPad (Siri Shortcuts), MacBook (Raycast/AppleScript), 결제 알림 |
| 알람 | Notion Calendar 앱 (확정 일정 자동 알람) + ntfy (AI 판단 알람) |
| 외부 접근 | Cloudflare Tunnel |

### 1-3. 핵심 전제 조건

- `AI 비서 일정` DB가 일정의 유일한 원천이다. 이 DB에 없는 시간은 빈 시간으로 간주한다.
- 승인 전에는 Notion을 수정하지 않는다.
- `실행 시간`에 시작·종료 시각이 모두 포함되어야 Notion Calendar 알람이 작동한다.
- `확정` 상태 일정만 시간 점유로 본다. `완료`, `취소`, `제안됨`은 빈 시간으로 계산한다.

## 2. 전체 시스템 아키텍처

### 2-1. 레이어 다이어그램

```
[입력 레이어]
  iPhone / iPad    → Siri Shortcuts → HTTP POST
  MacBook          → Raycast / AppleScript
  결제 알림        → 카드사 이메일(Gmail) / 문자(Siri Shortcuts)
  Windows 서버     → 매일 07:00 자동 실행 (아침 브리핑)
        ↓
[AI 엔진 — Python 서버 on Windows]
  Claude API
  ├── 목표 해석 & 하위 계획 생성
  ├── 일정 재조율 판단
  ├── 빈 시간 계산
  ├── 가계부 파싱
  └── 상황 인식 알람 판단
        ↓
[출력 레이어]
  Notion API
  ├── AI 비서 일정 DB  → Notion Calendar (확정 일정 자동 알람)
  └── 가계부 DB        → 지출 기록
  ntfy (Docker)        → AI 판단 기반 스마트 알람 → iPhone 푸시
```

### 2-2. 연결 기술

| 기술 | 역할 |
|---|---|
| Claude API | 모든 AI 판단의 중심 |
| Notion API | 일정 DB / 가계부 DB 읽기·쓰기 |
| Notion Calendar 앱 | `실행 시간` 기준 확정 일정 자동 알람 |
| Siri Shortcuts | 음성 → 텍스트 → 서버 HTTP POST |
| Flask | Python 서버 HTTP 엔드포인트 |
| ntfy (Docker) | AI 판단 스마트 알람 iPhone 푸시 |
| Cloudflare Tunnel | 서버 외부 접근 (무료, 주소 고정) |
| Gmail / Make.com | 결제 영수증 파싱 파이프라인 |

### 2-3. 데이터 흐름 원칙

- **읽기**: 항상 `AI 비서 일정` DB를 원천으로 한다.
- **쓰기**: 사용자 승인 후에만 Notion에 기록한다.
- **알람**: `확정` 저장 즉시 Notion Calendar가 자동 처리. AI 판단 알람은 ntfy 별도 발송.

## 4. 기능별 상세 흐름

### 4-1. 목표 → 일정 자동 생성

#### ① 사용자 입력

자연어로 큰 목표와 마감일을 말한다.

```
"다음 주 금요일까지 토스 면접 준비해야 해"
```

#### ② 목표 해석

AI가 추출하는 정보: 목표명 / 작업 유형 / 마감일 / 총 소요 시간(미정) / 포함 범위(미정)

#### ③ 과거 기록 확인

`AI 비서 일정` DB에서 `상태 = 완료` + 동일 `작업 유형`의 유사 기록을 조회한다.

| 기록 수 | 처리 방식 |
|---|---|
| 3개 이상 | 최근 3개 `실제 소요 시간(분)` 평균 사용 |
| 1~2개 | 있는 기록만 사용, "기록이 적다"고 근거에 표시 |
| 0개 | 사용자에게 질문 1개만: "이번 준비는 총 얼마나 걸릴 것 같아요?" |

> `실제 소요 시간(분)` 값이 없으면 `실행 시간`의 시작~종료 길이를 대신 사용한다.

#### ④ 하위 계획 생성

총 시간을 단순 블록으로 나누지 않는다. 반드시 의미 있는 하위 작업으로 구성한다.

예시별 하위 계획:

**면접 준비**
| 순서 | 하위 계획 | 목적 |
|---:|---|---|
| 1 | 예상 질문 정리 | 회사/직무/경험 기반 질문 후보 수집 |
| 2 | 답변 초안 작성 | STAR 구조 또는 핵심 bullet 작성 |
| 3 | 소리내어 연습 | 실제 말하기 기준으로 어색한 부분 수정 |
| 4 | 최종 점검 | 핵심 답변, 질문 리스트, 준비물 확인 |

**자소서**
| 순서 | 하위 계획 | 목적 |
|---:|---|---|
| 1 | 문항 요구사항 분석 | 질문이 묻는 평가 기준 파악 |
| 2 | 소재 선정 | 경험 후보를 고르고 하나로 좁힘 |
| 3 | 초안 작성 | 분량을 채우는 1차 작성 |
| 4 | 퇴고 | 문장, 흐름, 중복 제거 |

**공부**
| 순서 | 하위 계획 | 목적 |
|---:|---|---|
| 1 | 범위 확인 | 봐야 할 챕터나 자료 목록 정리 |
| 2 | 1차 학습 | 핵심 개념 이해 |
| 3 | 문제 풀이 또는 복습 | 기억 점검 |
| 4 | 약점 보완 | 틀린 부분 재학습 |

#### ⑤ 빈 시간 계산

**시간 점유 기준:**
- `상태 = 확정` + `실행 시간` 있는 일정만 점유
- `완료` / `취소` / `제안됨` 은 빈 시간으로 계산

**배치 규칙:**
- 세션 하나 최대 **2시간**
- 세션 사이 최소 **30분** 휴식
- 마감 **하루 전**까지 완료 우선 (시간 부족 시 마감 당일도 사용)
- 하루 총 작업 6시간 초과 → 경고
- 하루 총 작업 8시간 초과 → 재확인 요청
- 시간 부족 시 거짓 계획 대신 부족 시간 명시

#### ⑥ 제안 출력 (승인 전 Notion 미기록)

```
토스 면접 준비를 총 5시간 30분 기준으로 잡을게요.
최근 면접 준비 기록 3개 평균 5시간 20분 → 10분 올림했습니다.

1. 예상 질문 정리   | 2026-04-21 20:00~21:30
2. 답변 초안 작성   | 2026-04-22 20:00~21:30
3. 소리내어 연습    | 2026-04-23 19:30~21:00
4. 최종 점검        | 2026-04-24 10:00~11:00

이렇게 등록할까요?
```

시간 부족 시:

```
요청한 작업량은 12시간입니다.
마감 전 배치 가능한 시간은 5시간입니다.
부족한 시간은 7시간입니다.

가능한 만큼만 아래처럼 배치할 수 있어요.

1. 예상 질문 정리   | 2026-04-21 20:00~22:00
2. 답변 초안 작성   | 2026-04-22 19:00~21:00
3. 최종 점검        | 2026-04-23 09:00~10:00

전체 목표를 끝내기엔 시간이 부족합니다.
가능한 5시간만 등록할까요?
```

#### ⑦ 승인 후 Notion 생성

각 세션을 독립 일정으로 생성. 상태는 `확정`으로 저장 → Notion Calendar 자동 알람 활성화.

생성 시 각 일정 속성 예시:

| 속성 | 값 |
|---|---|
| 일정 이름 | 토스 면접 준비 - 예상 질문 정리 |
| 실행 시간 | 2026-04-21 20:00~21:30 |
| 작업 유형 | 면접 |
| 상태 | 확정 |
| 우선순위 | 보통 |
| 마감일 | 2026-04-25 |
| 원래 목표 | 다음 주 금요일까지 토스 면접 준비해야 해 |
| 하위 계획 | 예상 질문 정리 |
| 추정 근거 | 최근 면접 준비 기록 3개 평균 5시간 20분, 5시간 30분으로 올림 |
| AI 생성 여부 | true |
| 설명 | 회사/직무/경험 기반 예상 질문 후보 수집 |

### 4-2. 일정 변경 & 자동 재조율

#### 흐름

1. 사용자가 변경 사유를 자연어로 말한다.
   ```
   "오늘 저녁에 갑자기 술 약속 생겼어"
   ```
2. AI가 충돌하는 `확정` 일정을 감지한다.
3. 마감일과 우선순위를 고려해 밀 수 있는 빈 슬롯을 탐색한다.
4. 마감 초과 위험이 있으면 경고와 함께 재조율 안을 제안한다.
5. 승인 시 Notion 일정 `실행 시간` 업데이트.

**역질문 규칙:** 정보 부족 시 질문 1개만. 이미 충분하면 바로 제안.

#### 대화 예시

```
나:  오늘 저녁에 갑자기 술 약속 생겼어
AI:  오늘 저녁 7시에 알고리즘 과제가 잡혀 있어요.
     마감이 내일 오전 10시인데, 오늘 오전 11시~오후 1시 슬롯이 비어 있어요.
     알고리즘 과제를 오전 11시~오후 1시로 옮기고, 저녁엔 술 약속을 넣을까요?

나:  ㅇㅇ 그렇게 해줘
AI:  알고리즘 과제 → 오늘 오전 11시~오후 1시로 이동
     술 약속 → 오늘 오후 7시 추가
     Notion에 반영할게요.
```

### 4-3. 결제 자동 가계부

#### 구현 방법 (우선순위 순)

| 순위 | 방법 | 파이프라인 |
|---|---|---|
| 1 | 카드사 이메일 영수증 파싱 | Gmail → Make.com → Claude 파싱 → Notion |
| 2 | 결제 문자 파싱 | Siri Shortcuts 감지 → 서버 → Notion |
| 3 | 앱 푸시 파싱 | 제한 많아 후순위 |

Claude가 파싱해 `가계부 DB`에 기록. 기록 항목: 이름 / 금액 / 카테고리 / 날짜 / 결제수단.

### 4-4. 상황 인식 스마트 알람

#### 매일 07:00 자동 실행 흐름

1. 당일 `확정` 일정 전체 조회
2. Claude가 각 일정의 내용을 판단
3. 특별히 챙길 것이 있으면 ntfy로 iPhone 푸시

#### 자동 알람 생성 기준

| 상황 | 알람 타이밍 | 내용 |
|---|---|---|
| 기차 / 비행기 / 버스 탑승 | 1시간 전 | "짐 다시 확인하세요" |
| 시험 / 제출 마감 당일 | 당일 아침 | "오늘 마감입니다" |
| 중요 미팅 | 30분 전 | 미팅 알림 |
| 약 복용 / 병원 예약 | 당일 아침 | 복용/예약 알림 |

#### Notion Calendar vs ntfy 역할 구분

| 채널 | 역할 |
|---|---|
| Notion Calendar 앱 | 모든 `확정` 일정 시작 전 기본 알람 (자동) |
| ntfy | AI가 맥락을 판단해 추가로 발송하는 스마트 알람 |

## 5. DB 스키마

### 5-1. AI 비서 일정 DB

| 속성 | 타입 | 설명 |
|---|---|---|
| 일정 이름 | Title | Notion Calendar에 보일 세션 이름 |
| 실행 시간 | Date | 시작·종료 시각 모두 포함 필수 (Calendar 알람 기준) |
| 작업 유형 | Select | 자소서 / 면접 / 공부 / 운동 / 약속 / 업무 / 기타 |
| 상태 | Status | 제안됨 / 확정 / 완료 / 취소 |
| 우선순위 | Select | 높음 / 보통 / 낮음 |
| 마감일 | Date | 목표의 실제 마감일. 캘린더 배치 기준 아님 |
| 원래 목표 | Text | 사용자가 처음 말한 문장 |
| 하위 계획 | Text | 이 세션이 전체 계획에서 담당하는 구체 작업 |
| 추정 근거 | Text | 최근 3개 평균 / 사용자 입력 등 소요 시간 근거 |
| 실제 소요 시간(분) | Number | 완료 후 수정 가능. 비어 있으면 `실행 시간` 길이 사용 |
| AI 생성 여부 | Checkbox | AI가 만든 일정인지 표시 |
| 설명 | Text | 준비물 / 주의사항 / 메모 |

**상태별 시간 점유 규칙:**

| 상태 | 점유 여부 |
|---|---|
| 확정 | 점유 (빈 시간 계산에서 제외) |
| 완료 | 빈 시간으로 계산 |
| 취소 | 빈 시간으로 계산 |
| 제안됨 | 빈 시간으로 계산 |

**Notion Calendar 연결:** `실행 시간` 속성에 연결. `마감일`은 AI 판단용, 캘린더 배치 기준 아님.

### 5-2. 가계부 DB

| 속성 | 타입 | 설명 |
|---|---|---|
| 이름 | Title | 결제 내용 |
| 금액 | Number | 결제 금액 (원) |
| 카테고리 | Select | 식비 / 교통 / 쇼핑 / 구독 / 기타 |
| 날짜 | Date | 결제 일시 |
| 결제수단 | Select | 카드 / 현금 / 계좌이체 |
| 메모 | Text | 추가 메모 |

## 6. 알림 시스템

| 채널 | 역할 | 조건 |
|---|---|---|
| Notion Calendar 앱 | 확정 일정 시작 전 기본 알람 | `상태 = 확정` + `실행 시간`에 시각 포함 시 자동 |
| ntfy (Docker) | AI 판단 기반 스마트 알람 | 매일 07:00 브리핑 또는 AI 판단 시 발송 |

**ntfy 설치 (Windows Docker Desktop):**

```powershell
docker run -p 80:80 -v ntfy-data:/var/lib/ntfy binwiederhier/ntfy serve
```

**외부 접근 (Cloudflare Tunnel):**

```powershell
cloudflared tunnel --url http://localhost:80
# → https://xxxx.cfargotunnel.com 고정 주소
```

## 7. 인프라 & 배포

### 7-1. 현재: Windows 데스크톱

**폴더 구조:**

```
C:\ai-assistant\
  ├── main.py          # 아침 브리핑 스케줄러 (07:00 자동 실행)
  ├── notion.py        # Notion API 함수 (일정 DB / 가계부 DB)
  ├── notify.py        # ntfy 알림 함수
  ├── claude.py        # Claude API + 시스템 프롬프트 정의
  ├── scheduler.py     # 일정 분석 & 재조율 로직
  ├── server.py        # Flask HTTP 엔드포인트 (Siri Shortcuts 수신)
  ├── .env             # API 키 보관 (git 제외 필수)
  └── requirements.txt
```

**필요 패키지:** `anthropic`, `notion-client`, `requests`, `python-dotenv`, `schedule`, `flask`

**환경 변수 (.env):**

```
ANTHROPIC_API_KEY=sk-ant-...
NOTION_API_KEY=secret_...
NOTION_SCHEDULE_DB_ID=...
NOTION_LEDGER_DB_ID=...
NTFY_URL=https://xxxx.cfargotunnel.com/내채널
```

Windows 작업 스케줄러로 부팅 시 자동 실행 등록.

### 7-2. 추후: Mac Mini 이전

변경 필요한 것: **launchd plist 등록 하나뿐.** 코드 / `.env` / Docker(ntfy) / Cloudflare Tunnel 설정은 100% 재사용 가능.

추가로 가능해지는 것:
- AppleScript로 iPhone 캘린더 / 미리알림 앱 직접 제어
- Siri Shortcuts → 로컬 네트워크로 Mac Mini 서버 직접 호출

## 9. 주의사항 & 팁

- `.env` 파일은 `.gitignore`에 반드시 추가. Notion API 키는 해당 DB에만 권한 부여.
- Notion 날짜 형식: ISO 8601 한국 시각 기준 (`"2026-04-22T21:00:00+09:00"`).
- `실행 시간`에 시각이 없으면 Notion Calendar 알람이 작동하지 않는다. Claude 출력 시 시각 포함을 시스템 프롬프트에 반드시 명시해야 한다.
- 대화 히스토리는 최근 20개 메시지만 유지 (API 비용 절감). 일정 등록 완료 후 초기화.
- ntfy 채널명은 추측하기 어려운 이름 사용 (예: `my-assistant-a7f3k`).
- 하루 총 작업 6시간 초과 시 경고, 8시간 초과 시 재확인 요청. AI는 거짓 계획을 만들지 않고 부족 시간을 명시한다.
